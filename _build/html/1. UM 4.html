Klasyfikacja#

Wprowadzenie do klasyfikacji#

Klasyfikacja to jedno z kluczowych zada≈Ñ uczenia maszynowego, polegajƒÖce na przypisywaniu nowych danych do jednej lub wiƒôcej zdefiniowanych kategorii. Jest szeroko stosowana w r√≥≈ºnych dziedzinach, takich jak analiza tekstu, rozpoznawanie obraz√≥w, medycyna czy prognozowanie finansowe.

Czym jest klasyfikacja?#

Klasyfikacja polega na opracowaniu funkcji decyzyjnej, kt√≥ra na podstawie zestawu cech wej≈õciowych okre≈õla, do kt√≥rej kategorii (klasy) nale≈ºy dany obiekt. Algorytm uczy siƒô tej funkcji na podstawie oznaczonych danych treningowych, kt√≥re zawierajƒÖ przyk≈Çady z przypisanymi etykietami klas.

Rodzaje klasyfikacji#

  1. Klasyfikacja binarna:

    • Dane sƒÖ przypisywane do jednej z dw√≥ch klas.

    • Przyk≈Çad: Czy email jest spamem? (Spam/nie-spam)

  2. Klasyfikacja wieloklasowa:

    • Dane mogƒÖ nale≈ºeƒá do jednej z wielu klas.

    • Przyk≈Çad: Rozpoznawanie gatunk√≥w kwiat√≥w (setosa, versicolor, virginica).

  3. Klasyfikacja wieloetykietowa:

    • Dane mogƒÖ nale≈ºeƒá do wiƒôcej ni≈º jednej klasy jednocze≈õnie.

    • Przyk≈Çad: Oznaczanie temat√≥w w artyku≈Çach (sport, technologia, polityka).

Przyk≈Çady zastosowa≈Ñ klasyfikacji#

  1. Medycyna:

    • Diagnoza chor√≥b na podstawie wynik√≥w bada≈Ñ (zdrowy/chory).

  2. Analiza tekstu:

    • Klasyfikacja wiadomo≈õci email (spam/nie-spam).

    • Analiza sentymentu (pozytywny/negatywny).

  3. Rozpoznawanie obraz√≥w:

    • Identyfikacja obiekt√≥w na zdjƒôciach (kot/pies).

  4. Finanse:

    • Przewidywanie ryzyka kredytowego (dobra/z≈Ça zdolno≈õƒá kredytowa).

  5. In≈ºynieria i przemys≈Ç:

    • Wykrywanie usterek w maszynach (dzia≈Ça/nie dzia≈Ça).

Jak dzia≈Ça klasyfikacja?#

  1. Zbi√≥r danych treningowych:

    • Algorytm otrzymuje zbi√≥r przyk≈Çad√≥w, gdzie ka≈ºda pr√≥bka jest oznaczona etykietƒÖ klasy.

  2. Budowa modelu:

    • Algorytm tworzy regu≈Çy (funkcjƒô decyzyjnƒÖ), kt√≥re pozwalajƒÖ przypisaƒá nowe dane do odpowiednich klas.

  3. Ocena modelu:

    • Model jest testowany na danych testowych, kt√≥re nie by≈Çy u≈ºywane podczas treningu.

  4. Przewidywanie:

    • Model przypisuje nowe pr√≥bki do odpowiednich klas na podstawie wyuczonych regu≈Ç.

Podstawowe algorytmy klasyfikacyjne#

  1. Regresja logistyczna:

    • Klasyczny algorytm do binarnej klasyfikacji, kt√≥ry przewiduje prawdopodobie≈Ñstwo przynale≈ºno≈õci do danej klasy.

  2. k-Nearest Neighbors (k-NN):

    • Algorytm oparty na odleg≈Ço≈õciach; przypisuje klasƒô na podstawie najbli≈ºszych sƒÖsiad√≥w w przestrzeni cech.

  3. Drzewa decyzyjne:

    • TworzƒÖ strukturƒô drzewa, gdzie ka≈ºdy wƒôze≈Ç reprezentuje decyzjƒô opartƒÖ na cechach danych.

  4. Naive Bayes:

    • Algorytm probabilistyczny zak≈ÇadajƒÖcy niezale≈ºno≈õƒá cech; czƒôsto u≈ºywany w klasyfikacji tekstu.

  5. Support Vector Machines (SVM):

    • Algorytm, kt√≥ry maksymalizuje margines miƒôdzy klasami, ≈õwietnie sprawdza siƒô w przypadku z≈Ço≈ºonych granic decyzyjnych.

Granica decyzyjna#

Granica decyzyjna to linia (lub powierzchnia w wielowymiarowej przestrzeni), kt√≥ra oddziela r√≥≈ºne klasy. W klasyfikacji kluczowe jest, aby granica decyzyjna by≈Ça dobrze dopasowana do danych:

  • Overfitting: Granica zbyt dok≈Çadnie odzwierciedla dane treningowe, przez co model nie generalizuje.

  • Underfitting: Granica jest zbyt prosta, przez co model ignoruje wa≈ºne wzorce w danych.

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
from sklearn.model_selection import train_test_split
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score
import ipywidgets as widgets
from ipywidgets import interact

# Generate synthetic dataset
X, y = make_moons(n_samples=300, noise=0.1, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Interactive function for classification
def interactive_classification(C=1.0, kernel='rbf', gamma=1.0):
    # Train SVM model with specified parameters
    model = SVC(C=C, kernel=kernel, gamma=gamma)
    model.fit(X_train, y_train)
    
    # Predict and calculate accuracy
    y_pred_train = model.predict(X_train)
    y_pred_test = model.predict(X_test)
    train_acc = accuracy_score(y_train, y_pred_train)
    test_acc = accuracy_score(y_test, y_pred_test)
    
    # Plot decision boundary
    plt.figure(figsize=(12, 6))
    
    # Create mesh grid for decision boundary
    x_min, x_max = X[:, 0].min() - 0.5, X[:, 0].max() + 0.5
    y_min, y_max = X[:, 1].min() - 0.5, X[:, 1].max() + 0.5
    xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.01), np.arange(y_min, y_max, 0.01))
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    
    # Plot decision boundary and data points
    plt.contourf(xx, yy, Z, alpha=0.8, cmap=plt.cm.Paired)
    plt.scatter(X_train[:, 0], X_train[:, 1], c=y_train, marker='o', label='Training Data', edgecolor='k')
    plt.scatter(X_test[:, 0], X_test[:, 1], c=y_test, marker='s', label='Test Data', edgecolor='k')
    
    # Titles and labels
    plt.title(f"SVM Decision Boundary\nTrain Accuracy: {train_acc:.2f}, Test Accuracy: {test_acc:.2f}")
    plt.xlabel("Feature 1")
    plt.ylabel("Feature 2")
    plt.legend()
    plt.grid(True)
    plt.show()

# Interactive widgets for parameters
interact(
    interactive_classification,
    C=widgets.FloatLogSlider(value=1.0, base=10, min=-2, max=2, step=0.1, description='C (Regularization)'),
    kernel=widgets.RadioButtons(options=['linear', 'rbf', 'poly'], value='rbf', description='Kernel'),
    gamma=widgets.FloatLogSlider(value=1.0, base=10, min=-2, max=2, step=0.1, description='Gamma')
)

Jak algorytmy klasyfikacyjne uczƒÖ siƒô na przyk≈Çadzie regresji logistycznej#

Algorytmy klasyfikacyjne uczƒÖ siƒô przez proces optymalizacji, kt√≥rego celem jest zminimalizowanie b≈Çƒôdu predykcji. Na przyk≈Çadzie regresji logistycznej mo≈ºemy prze≈õledziƒá, jak dzia≈Ça proces uczenia i jakie kroki sƒÖ podejmowane, aby model poprawnie klasyfikowa≈Ç dane.

1. Funkcja decyzyjna w regresji logistycznej#

Regresja logistyczna to algorytm klasyfikacyjny stosowany do zada≈Ñ binarnej klasyfikacji. Funkcja decyzyjna jest okre≈õlona jako:

\[ P(y=1 | x) = \sigma(w^T x + b) \]

gdzie:

  • \( \sigma(z) = \frac{1}{1 + e^{-z}} \) ‚Äì funkcja sigmoid (przekszta≈Çca wyniki w zakres \( (0, 1) \)),

  • \( w \) ‚Äì wektor wag,

  • \( x \) ‚Äì wektor cech,

  • \( b \) ‚Äì bias (przesuniƒôcie).

Funkcja sigmoid zwraca prawdopodobie≈Ñstwo przynale≈ºno≈õci pr√≥bki do klasy \( y=1 \).

2. Funkcja kosztu (Log-Loss)#

Aby nauczyƒá model odpowiednich warto≈õci wag \( w \) i biasu \( b \), nale≈ºy zdefiniowaƒá funkcjƒô kosztu, kt√≥ra mierzy, jak dobrze model dopasowuje siƒô do danych treningowych. W regresji logistycznej funkcjƒÖ kosztu jest logarytmiczna funkcja straty (log-loss):

\[ \mathcal{L}(w, b) = -\frac{1}{N} \sum_{i=1}^N \left[ y_i \log(\hat{y}_i) + (1 - y_i) \log(1 - \hat{y}_i) \right] \]

gdzie:

  • \( y_i \) ‚Äì prawdziwa etykieta dla pr√≥bki \( i \),

  • \( \hat{y}_i = P(y=1 | x_i) \) ‚Äì przewidywane prawdopodobie≈Ñstwo przez model,

  • \( N \) ‚Äì liczba pr√≥bek w zbiorze treningowym.

Funkcja kosztu karze za b≈Çƒôdne przewidywania, szczeg√≥lnie te o wysokim poziomie pewno≈õci.

3. Optymalizacja funkcji kosztu#

Uczenie siƒô modelu polega na minimalizacji funkcji kosztu. Proces optymalizacji sk≈Çada siƒô z nastƒôpujƒÖcych krok√≥w:

Krok 1: Inicjalizacja parametr√≥w#

  • PoczƒÖtkowe warto≈õci wag \( w \) i biasu \( b \) sƒÖ ustawiane losowo lub na zero.

Krok 2: Obliczanie gradientu#

  • Gradient to wektor pochodnych funkcji kosztu wzglƒôdem parametr√≥w \( w \) i \( b \): $\( \frac{\partial \mathcal{L}}{\partial w_j} = \frac{1}{N} \sum_{i=1}^N ( \hat{y}_i - y_i ) x_{ij} \)\( \)\( \frac{\partial \mathcal{L}}{\partial b} = \frac{1}{N} \sum_{i=1}^N ( \hat{y}_i - y_i ) \)$

  • Gradient wskazuje kierunek najwiƒôkszego wzrostu funkcji kosztu.

Krok 3: Aktualizacja parametr√≥w#

  • Parametry sƒÖ aktualizowane w kierunku przeciwnym do gradientu, aby minimalizowaƒá funkcjƒô kosztu. Jest to realizowane za pomocƒÖ algorytmu gradient descent: $\( w_j := w_j - \eta \frac{\partial \mathcal{L}}{\partial w_j} \)\( \)\( b := b - \eta \frac{\partial \mathcal{L}}{\partial b} \)\( gdzie \) \eta $ to wsp√≥≈Çczynnik uczenia (ang. learning rate).

Krok 4: Iteracje#

  • Proces obliczania gradientu i aktualizacji parametr√≥w jest powtarzany przez wiele iteracji (epok), a≈º do osiƒÖgniƒôcia minimalnej warto≈õci funkcji kosztu lub spe≈Çnienia innego kryterium stopu.

4. Wp≈Çyw hiperparametr√≥w funkcji optymalizacyjnej#

Wsp√≥≈Çczynnik uczenia (\( \eta \))#

  • Zbyt ma≈Çy: Proces uczenia jest wolny, mo≈ºe utknƒÖƒá w minimum lokalnym.

  • Zbyt du≈ºy: Model mo≈ºe przeskakiwaƒá minimum globalne, co prowadzi do niestabilno≈õci.

Liczba iteracji#

  • Zbyt ma≈Ça liczba iteracji powoduje niedouczenie modelu (ang. underfitting).

  • Zbyt du≈ºa liczba iteracji mo≈ºe prowadziƒá do przeuczenia modelu (ang. overfitting).

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score
import ipywidgets as widgets
from ipywidgets import interact

# Generate synthetic dataset
X, y = make_moons(n_samples=300, noise=0.25, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Interactive function for logistic regression with polynomial features
def interactive_logistic_regression(degree=1, C=1.0):
    # Add polynomial features
    poly = PolynomialFeatures(degree=degree)
    X_train_poly = poly.fit_transform(X_train)
    X_test_poly = poly.transform(X_test)
    
    # Standardize the data
    scaler = StandardScaler()
    X_train_poly = scaler.fit_transform(X_train_poly)
    X_test_poly = scaler.transform(X_test_poly)
    
    # Train logistic regression model
    model = LogisticRegression(C=C, max_iter=500, random_state=42)
    model.fit(X_train_poly, y_train)
    
    # Predict and calculate accuracy
    y_pred_train = model.predict(X_train_poly)
    y_pred_test = model.predict(X_test_poly)
    train_acc = accuracy_score(y_train, y_pred_train)
    test_acc = accuracy_score(y_test, y_pred_test)
    
    # Plot decision boundary
    plt.figure(figsize=(12, 6))
    
    # Create mesh grid for decision boundary
    x_min, x_max = X[:, 0].min() - 0.5, X[:, 0].max() + 0.5
    y_min, y_max = X[:, 1].min() - 0.5, X[:, 1].max() + 0.5
    xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.01), np.arange(y_min, y_max, 0.01))
    Z = model.predict(poly.transform(np.c_[xx.ravel(), yy.ravel()]))
    Z = Z.reshape(xx.shape)
    
    # Plot decision boundary and data points
    plt.contourf(xx, yy, Z, alpha=0.8, cmap=plt.cm.Paired)
    plt.scatter(X_train[:, 0], X_train[:, 1], c=y_train, marker='o', label='Training Data', edgecolor='k')
    plt.scatter(X_test[:, 0], X_test[:, 1], c=y_test, marker='s', label='Test Data', edgecolor='k')
    
    # Titles and labels
    plt.title(f"Logistic Regression with Polynomial Features (Degree={degree})\nTrain Accuracy: {train_acc:.2f}, Test Accuracy: {test_acc:.2f}")
    plt.xlabel("Feature 1")
    plt.ylabel("Feature 2")
    plt.legend()
    plt.grid(True)
    plt.show()

# Interactive widgets for parameters
interact(
    interactive_logistic_regression,
    degree=widgets.IntSlider(min=1, max=10, step=1, value=1, description='Degree'),
    C=widgets.FloatLogSlider(value=1.0, base=10, min=-2, max=2, step=0.1, description='C (Regularization)')
)

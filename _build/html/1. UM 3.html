Warto≈õci odstajƒÖce i ich wp≈Çyw na regresjƒô#

Warto≈õci odstajƒÖce to punkty danych, kt√≥re znaczƒÖco odbiegajƒÖ od og√≥lnego wzorca danych. W regresji majƒÖ szczeg√≥lne znaczenie, poniewa≈º mogƒÖ zniekszta≈Çcaƒá liniƒô najlepszego dopasowania, co mo≈ºe prowadziƒá do nieprecyzyjnych prognoz i b≈Çƒôdnych interpretacji.

Czym jest warto≈õƒá odstajƒÖca?#

Definicja: Warto≈õƒá odstajƒÖca to obserwacja, kt√≥ra le≈ºy daleko poza typowym zakresem warto≈õci w zestawie danych. Wyobra≈∫my sobie, ≈ºe badamy dzienne temperatury, a wiƒôkszo≈õƒá dni mie≈õci siƒô w zakresie 15‚Äì25¬∞C, ale pewnego dnia temperatura wzrasta do 40¬∞C. Ta warto≈õƒá 40¬∞C jest warto≈õciƒÖ odstajƒÖcƒÖ.

Dlaczego to wa≈ºne: W regresji warto≈õci odstajƒÖce mogƒÖ ‚ÄûprzeciƒÖgnƒÖƒá‚Äù liniƒô regresji w ich kierunku, co zniekszta≈Çca prognozy modelu. To mo≈ºe sprawiƒá, ≈ºe model bƒôdzie mniej skuteczny w przewidywaniu typowych punkt√≥w danych i bardziej podatny na losowy szum.

Jak radziƒá sobie z warto≈õciami odstajƒÖcymi?#

IstniejƒÖ trzy podstawowe podej≈õcia:#

Usu≈Ñ warto≈õci odstajƒÖce: W przypadkach, gdy warto≈õci odstajƒÖce wynikajƒÖ z b≈Çƒôd√≥w w danych lub ekstremalnych wydarze≈Ñ, kt√≥re raczej siƒô nie powt√≥rzƒÖ, ich usuniƒôcie mo≈ºe poprawiƒá dok≈Çadno≈õƒá modelu.

U≈ºyj regresji odpornej: Niekt√≥re modele, takie jak regresja odporna, sƒÖ mniej wra≈ºliwe na warto≈õci odstajƒÖce i minimalizujƒÖ ich wp≈Çyw na liniƒô najlepszego dopasowania.

Przekszta≈Çƒá dane: Zastosowanie przekszta≈Çce≈Ñ (np. przekszta≈Çcenie logarytmiczne) mo≈ºe pom√≥c zmniejszyƒá wp≈Çyw warto≈õci odstajƒÖcych poprzez ‚Äûskurczenie‚Äù du≈ºych odchyle≈Ñ. Usuwanie warto≈õci odstajƒÖcych?

TypowƒÖ metodƒÖ radzenia sobie z warto≈õciami odstajƒÖcymi jest prosta filtracja oparta na rozk≈Çadzie danych. Pamiƒôtasz rozk≈Çady?

# Import necessary libraries
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression
import ipywidgets as widgets
from scipy.stats import gaussian_kde
from IPython.display import display

# Set random seed for reproducibility
np.random.seed(42)

# Generate synthetic data with a linear trend and add some controlled outliers
X = np.arange(1, 31).reshape(-1, 1)  # Feature (1, 2, 3, ..., 30)
y = 2.5 * X.squeeze() + np.random.normal(0, 3, X.shape[0])  # Linear relationship with noise

# Introduce controlled outliers
y[7] += 25
y[14] -= 20
y[21] += 30

# Function to plot regression with quantile-based outlier filtering and KDE plot
def plot_filtered_regression(quantile_threshold=0.05):
    # Calculate quantile threshold limits for filtering
    lower_limit = np.quantile(y, quantile_threshold)
    upper_limit = np.quantile(y, 1 - quantile_threshold)
    
    # Filter data based on quantile threshold limits
    mask = (y >= lower_limit) & (y <= upper_limit)
    X_filtered = X[mask]
    y_filtered = y[mask]
    
    # Plot KDE and quantile threshold lines
    plt.figure(figsize=(10, 5))
    y_range = np.linspace(min(y) - 5, max(y) + 5, 1000)
    kde = gaussian_kde(y, bw_method='scott')
    kde_values = kde(y_range)
    plt.plot(y_range, kde_values, label="KDE of y", color="blue")
    plt.axvline(x=lower_limit, color="red", linestyle="--", label=f"Lower Threshold (Quantile {quantile_threshold})")
    plt.axvline(x=upper_limit, color="red", linestyle="--", label=f"Upper Threshold (Quantile {1 - quantile_threshold})")
    plt.xlabel("y (Target Values)")
    plt.ylabel("Density")
    plt.title("KDE Plot of y with Quantile Thresholds")
    plt.legend()
    plt.show()
    
    # Check if there are enough points left after filtering
    if len(X_filtered) > 1:
        # Fit regression models for comparison
        model_original = LinearRegression().fit(X, y)
        y_pred_original = model_original.predict(X)
        
        model_filtered = LinearRegression().fit(X_filtered, y_filtered)
        y_pred_filtered = model_filtered.predict(X_filtered)
        
        # Plot the original and filtered regression lines
        plt.figure(figsize=(10, 6))
        plt.scatter(X, y, color="blue", label="Original Data with Outliers")
        plt.scatter(X_filtered, y_filtered, color="orange", label="Filtered Data (No Outliers)")
        plt.plot(X, y_pred_original, color="red", linestyle="--", linewidth=2, label="Regression Line (Original)")
        plt.plot(X_filtered, y_pred_filtered, color="green", linewidth=2, label="Regression Line (Filtered)")
        plt.xlabel("Feature (X)")
        plt.ylabel("Target (y)")
        plt.title("Outlier Removal using Quantile-based Filtering")
        plt.legend()
        plt.show()
    else:
        # Message if no data points are left after filtering
        print("No data points left after filtering. Adjust the threshold.")

# Interactive widget for setting quantile threshold
quantile_slider = widgets.FloatSlider(value=0.05, min=0.01, max=0.2, step=0.01, description="Quantile Threshold")

# Display the interactive plot
display(widgets.interactive(plot_filtered_regression, quantile_threshold=quantile_slider))

# Import necessary libraries

from IPython.display import display

# Set random seed for reproducibility
np.random.seed(42)

y[7] += 25
y[1] -= 25
y[14] -= 30
y[21] += 50

# Dictionary of available regression models with regularization
models = {
    "Linear Regression": LinearRegression(),
    "Ridge Regression (L2)": Ridge(alpha=14.0),
    "Lasso Regression (L1)": Lasso(alpha=14.0),
    "Elastic Net": ElasticNet(alpha=14.0, l1_ratio=0.5)
}

# Function to fit and visualize different regularization models
def plot_regularization(model_name="Linear Regression"):
    # Get the selected model from the dictionary
    model = models[model_name]
    
    # Fit the model to the data
    model.fit(X, y)
    y_pred = model.predict(X)
    
    # Fit the standard Linear Regression model for comparison
    linear_model = LinearRegression().fit(X, y)
    y_pred_linear = linear_model.predict(X)
    
    # Plotting the data and the regression lines
    plt.figure(figsize=(12, 6))
    plt.scatter(X, y, color="blue", label="Original Data with Outliers")
    plt.plot(X, y_pred_linear, color="red", linestyle="--", linewidth=2, label="Standard Linear Regression")
    plt.plot(X, y_pred, color="green", linewidth=2, label=f"{model_name}")
    plt.xlabel("Feature (X)")
    plt.ylabel("Target (y)")
    plt.title(f"Comparison of {model_name} with Standard Linear Regression")
    plt.legend()
    plt.show()

# Interactive widget for selecting regularization model
model_selector = widgets.Dropdown(
    options=["Linear Regression", "Ridge Regression (L2)", "Lasso Regression (L1)", "Elastic Net"],
    value="Linear Regression",
    description="Select Model:"
)

# Display the interactive plot
display(widgets.interactive(plot_regularization, model_name=model_selector))

Zaawansowany model regresji - regresja wielomianowa#

Chocia≈º regresja liniowa jest prosta i skuteczna, dane rzeczywiste sƒÖ czƒôsto zbyt z≈Ço≈ºone, aby dopasowaƒá do nich prostƒÖ liniƒô. Na szczƒô≈õcie istniejƒÖ bardziej zaawansowane modele regresji, kt√≥re potrafiƒÖ uchwyciƒá z≈Ço≈ºone wzorce w danych. Modele te pozwalajƒÖ uwzglƒôdniaƒá nieliniowe zale≈ºno≈õci, interakcje miƒôdzy cechami, a nawet dostosowywaƒá siƒô do sytuacji, gdy dane zawierajƒÖ warto≈õci odstajƒÖce.

Tutaj om√≥wimy trzy przyk≈Çady bardziej zaawansowanych technik regresji, kt√≥re wykraczajƒÖ poza regresjƒô liniowƒÖ:

Co to jest: Regresja wielomianowa rozszerza regresjƒô liniowƒÖ poprzez dodanie kombinacji oryginalnych cech. Zamiast dopasowywaƒá tylko liniƒô, model mo≈ºe dopasowaƒá krzywƒÖ, dodajƒÖc wyrazy podniesione do kwadratu (np. \(X^2\)) lub nawet wy≈ºsze potƒôgi.

Jak to dzia≈Ça: W regresji wielomianowej tworzymy nowe ‚Äûcechy‚Äù poprzez podnoszenie warto≈õci oryginalnych cech do wy≈ºszych potƒôg. Nowe cechy umo≈ºliwiajƒÖ modelowi wyginanie siƒô i dopasowanie do danych, kt√≥re podƒÖ≈ºajƒÖ za nieliniowym trendem.

    # Polynomial transformation

    X_poly = poly_features.fit_transform(X)
    X_test_poly = poly_features.transform(X_test)

    # Calculate performance metrics

    test_mse = mean_squared_error(y_test, y_pred_test)
    train_r2 = r2_score(y, y_pred_train)
    test_r2 = r2_score(y_test, y_pred_test)
    
    # Plotting the data and polynomial regression fit
    plt.figure(figsize=(10, 6))

Aby poradziƒá sobie z problemem niedouczenia i przeuczenia, mo≈ºna zastosowaƒá kilka technik:

Regularyzacja#

Co to jest: Regularyzacja dodaje karƒô dla du≈ºych wsp√≥≈Çczynnik√≥w w modelu, co pomaga zmniejszyƒá z≈Ço≈ºono≈õƒá modelu i uniknƒÖƒá przeuczenia.

Techniki: Regresja Ridge Regression oraz regresja Lasso to przyk≈Çady modeli z regularyzacjƒÖ, kt√≥re mogƒÖ pom√≥c w kontrolowaniu z≈Ço≈ºono≈õci modelu.

Wyb√≥r odpowiedniego modelu#

Co to oznacza: Wyb√≥r zbyt prostego modelu (jak regresja liniowa dla nieliniowych danych) mo≈ºe prowadziƒá do niedouczenia, a zbyt z≈Ço≈ºonego (jak regresja wielomianowa wysokiego stopnia) ‚Äì do przeuczenia.

Jak wybraƒá: Testowanie r√≥≈ºnych modeli i ocena ich wydajno≈õci za pomocƒÖ metryk, takich jak b≈ÇƒÖd ≈õredniokwadratowy (MSE) lub wsp√≥≈Çczynnik determinacji R2R2, mogƒÖ pom√≥c znale≈∫ƒá odpowiedni model dla danych.

Zastosowanie sprawdzianu krzy≈ºowego#

Co to jest: Kroswalidacja pozwala na ocenƒô modelu na r√≥≈ºnych podzbiorach danych, co daje bardziej wiarygodne wyniki ni≈º pojedynczy podzia≈Ç na dane treningowe i testowe.

Zalety: Pomaga w ocenie stabilno≈õci modelu i mo≈ºe wskazaƒá, czy model jest podatny na przeuczenie.

Zwiƒôkszenie ilo≈õci danych#

Dlaczego: Wiƒôksza liczba danych mo≈ºe pom√≥c modelowi nauczyƒá siƒô og√≥lnych wzorc√≥w i byƒá mniej podatnym na przeuczenie.

Jak to zrobiƒá: Mo≈ºna spr√≥bowaƒá zebraƒá wiƒôcej danych, wzbogaciƒá istniejƒÖce dane, lub zastosowaƒá techniki takie jak augmentacja danych w kontek≈õcie danych obrazowych.

Transformacja cech#

Dlaczego: Zastosowanie odpowiednich transformacji (np. logarytmicznych lub pierwiastkowych) mo≈ºe pom√≥c w lepszym dopasowaniu modelu do danych i zmniejszeniu wp≈Çywu warto≈õci odstajƒÖcych.

Jak: Na przyk≈Çad, zastosowanie logarytm√≥w do wybranych cech w regresji wielomianowej mo≈ºe pom√≥c modelowi lepiej uchwyciƒá nieliniowe zale≈ºno≈õci.

Stosowanie tych metod mo≈ºe pom√≥c w znalezieniu modelu, kt√≥ry dobrze dopasowuje siƒô do danych, minimalizujƒÖc jednocze≈õnie ryzyko niedouczenia i przeuczenia.

Sprawdzian krzy≈ºowy#

Sprawdzian krzy≈ºowy (ang. cross-validation) to technika stosowana w uczeniu maszynowym, kt√≥ra pozwala na lepszƒÖ ocenƒô jako≈õci modelu poprzez podzia≈Ç danych na wiele r√≥≈ºnych zestaw√≥w:

UczƒÖcy
Walidacyjny
Testowy

Dziƒôki temu mo≈ºemy uzyskaƒá bardziej wiarygodne wyniki ni≈º w przypadku pojedynczego podzia≈Çu danych na zbi√≥r treningowy i testowy.

Zbi√≥r uczƒÖcy wykorzystywany jest podczas trenowania algorymtu, czyli podczas wpasowywania funkcji regresji

Zbi√≥r walidacyjny jest wykorzystywany podczas test√≥w algorytmu, czyli podczas oceny wstƒôpnej naszego algorytmu

Zbi√≥r testowy wykorzystywany jest tylko raz do ostatecznego testu i oceny naszego algorytmu

Jak dzia≈Ça sprawdzian krzy≈ºowy?#

NajpopularniejszƒÖ metodƒÖ jest k-krotny sprawdzian krzy≈ºowy (k-fold cross-validation). W tej metodzie wykonuje siƒô:

Podzia≈Ç danych: Dane sƒÖ dzielone na kk r√≥wnych czƒô≈õci, czyli ‚Äûfold√≥w‚Äù (zwykle wybiera siƒô k=5 lub k=10).

Trenowanie i testowanie: Model jest trenowany k razy, za ka≈ºdym razem na innym podzbiorze danych. W ka≈ºdej iteracji jedna z czƒô≈õci jest u≈ºywana jako zbi√≥r testowy, a pozosta≈Çe $k‚àí1$ czƒô≈õci sƒÖ u≈ºywane do trenowania modelu.

≈örednia wynik√≥w: Po k iteracjach uzyskujemy k wynik√≥w, kt√≥re nastƒôpnie u≈õredniamy, aby uzyskaƒá ko≈ÑcowƒÖ miarƒô jako≈õci modelu (np. ≈õredniƒÖ warto≈õƒá b≈Çƒôdu lub wsp√≥≈Çczynnik determinacji $R^2$).

Dlaczego warto stosowaƒá sprawdzian krzy≈ºowy?#

Kroswalidacja jest przydatna, poniewa≈º:

Zmniejsza ryzyko losowego b≈Çƒôdu: Testowanie modelu na r√≥≈ºnych podzbiorach danych pozwala uzyskaƒá bardziej stabilne i reprezentatywne wyniki.

Pomaga wykryƒá przeuczenie: Je≈õli model dobrze dzia≈Ça na treningowych podzbiorach, ale ≈∫le na testowych, mo≈ºe to byƒá oznaka przeuczenia.

W pe≈Çni wykorzystuje dane: Ka≈ºdy punkt danych jest wykorzystany zar√≥wno w treningu, jak i w testowaniu, co jest szczeg√≥lnie wa≈ºne, gdy mamy ma≈Çy zestaw danych.

Przyk≈Çad#

Metody nadzorowane ‚ÄúuczƒÖ siƒô‚Äù poznajƒÖc sw√≥j b≈ÇƒÖd (zatem uczƒÖ siƒô na b≈Çƒôdach) - dziƒôki przyk≈Çadom wiedzƒÖ jak bardzo siƒô mylƒÖ i co nale≈ºy zmieniƒá. Mo≈ºna to przyr√≥wnaƒá do nauki studenta:

Student siƒô uczy/studiuje robiƒÖc zadania w domu. Po wykonaniu zadania sprawdza odpowied≈∫ - je≈õli siƒô zgadza, wiedza jest utrwalana i student wie, ≈ºe robi zadanie poprawnie. Je≈õli nie, poprawia siƒô.

Student przychodzi na kolokwium. Tych pyta≈Ñ nigdy nie widzia≈Ç i nie zna odpowiedzi, przynajmniej nie od razu. Je≈õli p√≥jdzie mu dobrze, wie ≈ºe jest na dobrej drodze. Je≈õli nie - trzeba wr√≥ciƒá do domu i zn√≥w siƒô uczyƒá.

from sklearn.metrics import mean_squared_error

from ipywidgets import interact

# Generate sample data
np.random.seed(0)
X = np.sort(5 * np.random.rand(50, 1), axis=0)
y = np.sin(X*2).ravel() + np.random.normal(0, 0.4, X.shape[0])

# Split the data into training and validation sets
X_train, X_val, y_train, y_val = train_test_split(X, y, test_size=0.2, random_state=0)

# Function to fit polynomial regression and plot results
def plot_polynomial_regression(degree):
    # Set up polynomial features
    poly = PolynomialFeatures(degree)
    X_train_poly = poly.fit_transform(X_train)
    X_val_poly = poly.transform(X_val)

    # Train the polynomial regression model
    model = LinearRegression()
    model.fit(X_train_poly, y_train)
    
    # Generate predictions
    X_plot = np.linspace(0, 5, 100).reshape(-1, 1)
    X_plot_poly = poly.transform(X_plot)
    y_train_pred = model.predict(X_train_poly)
    y_val_pred = model.predict(X_val_poly)
    y_plot_pred = model.predict(X_plot_poly)
    
    # Calculate Mean Squared Errors
    train_mse = mean_squared_error(y_train, y_train_pred)
    val_mse = mean_squared_error(y_val, y_val_pred)

    # Plot the results
    fig, (ax_train, ax_val) = plt.subplots(1, 2, figsize=(10, 5))
    
    # Training set plot
    ax_train.scatter(X_train, y_train, color='blue', label='Training Data')
    ax_train.plot(X_plot, y_plot_pred, color='red', label=f'Polynomial Degree {degree}')
    ax_train.set_title(f"Training Data Fit\nMean Squared Error: {train_mse:.2f}")
    ax_train.set_xlabel("X")
    ax_train.set_ylabel("y")
    ax_train.legend()
    
    # Validation set plot
    ax_val.scatter(X_val, y_val, color='green', label='Validation Data')
    ax_val.plot(X_plot, y_plot_pred, color='red', label=f'Polynomial Degree {degree}')
    ax_val.set_title(f"Validation Data Fit\nMean Squared Error: {val_mse:.2f}")
    ax_val.set_xlabel("X")
    ax_val.set_ylabel("y")
    ax_val.legend()
    
    plt.show()

# Interactive widget for polynomial degree
interact(plot_polynomial_regression, degree=widgets.IntSlider(min=1, max=30, step=1, value=2))
